# 适配器标准化（最小知识原则）

本文档定义 `ai-jue` 的适配器能力地图、命名规范与降级策略。

## 1. 设计目标

- 统一配置入口：`ai.config.js`
- 统一资产工作区：`.ai/`
- 统一能力概念：尽量对齐主流 AI 工具既有习惯
- 统一术语：使用 `agents`，`subAgents` 仅作为历史兼容别名

## 2. 8 大能力地图

| 能力 | 规范目录/字段 | 说明 |
| --- | --- | --- |
| 系统上下文 | `AGENTS.md` / `prompts.agents` | 全局约束与项目上下文 |
| 规则 | `rules/` | 项目规则（可按路径/场景） |
| 命令 | `commands/` / `commands.*` | 自定义命令与触发词 |
| 技能 | `skills/` / `skills.*` | 可复用任务能力 |
| 代理 | `agents/` / `agents.*` | 专用代理能力单元 |
| 钩子 | `hooks/` / `hooks.*` | 生命周期事件与自动化 |
| MCP | `mcp.servers` | 外部工具接入协议 |
| 工具逃生舱 | `tools/<tool>/` / `tools.*` | 工具私有配置透传 |

## 3. 目录协议（Preset 与 `.ai` 同构）

```text
.ai/
├── AGENTS.md
├── skills/
├── commands/
├── rules/
├── agents/
├── hooks/
└── tools/
    ├── gemini/
    └── cursor/
```

同一套目录协议同时适用于：
- 本地项目资产（`.ai/`）
- npm 预设包（`jue-preset-*`）

## 4. 兼容策略（当前过渡期）

- 规范名：`agents`
- 兼容别名：`subAgents`
- 过渡要求：
  - 文档统一使用 `agents`
  - 实现层提供 `subAgents -> agents` 兼容映射
  - 在破坏性移除前提供迁移说明与兼容窗口

## 5. 适配器降级原则

当目标工具不支持某能力时，必须降级且可解释：

- 不静默丢弃能力
- 在目标产物中保留提示或替代表达
- 在 `validate`/`check` 中报告能力降级信息

示例：
- 工具不支持 Hooks：输出“手动执行建议”而非无输出
- 工具不支持 Agents：降级到规则/上下文文本并附带说明

## 6. 当前实现边界（文档口径）

当前版本中部分适配器仍使用历史字段 `subAgents`。这属于兼容期实现细节，不改变规范层统一命名：

- 对外文档与设计：`agents`
- 对内兼容与迁移：`subAgents`（逐步收敛）

后续实现改造应遵循：先补齐兼容层，再统一核心字段，再清理历史命名。
